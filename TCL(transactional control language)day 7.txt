TCL(transactional control language)
---------------------------------
the process of combining all related operations into a single unit and executing them according to the rule "either all operations should be succeed or none should be succeed".
it is called transaction management.

A transaction is considered a single unit of work and follows the principle 
"either all or none"

example:- funds transfer.

consider a money transfer process between two accounts.

1.debit amount from the senders account.
credit amount into receivers account.

Bothe operations should be executed as a single transaction.

if the amount is debited from the senders account successfully . but creating the recovers account fails. it may lead to the data inconsistemcy problems.

there are two types of transactions such as:
1-Local transaction:-
--------------------
in local transactions all operations of a transaction are executed in the same database.

example:-funds transfer from one account to account where both accounts belong to the same bank 

2.Global Transactions:-
-------------------
in global transactions operations of a transaction are executed on different database.
example:- funds transfer from one account to account where both accounts belong to the different bank.

ACID properties
================
Atomicity:- all or nothing
consistency:- the database remains in a valid state
isolation:--transactions do not interface with each other
durability:- committed changes are permanent

WHAT IS LOCKING:-
================
locking is a mechanism used by the database to control concurrent access data.

it ensures that multiple users access the same data simultaneously the database maintains consistency,integrity,and prevent conflicts.

1.shared lock:-
--------------
A shared lock allows a transaction to read date.
Multiple transactions can hold a shared lock on the same row.
No transaction can modify the row until all shared locks are released.

characteristics:- 
read only
multiple users can read simultaneously 
blocks updates and deletes.

2.exclusive lock:-
------------------
An exclusive lock allows a transaction to modify data.
while an exclusive lock exists.
No other transaction can read using a locking read or modify that row.
only the transaction holding the lock can update /delete the data .

characteristics:-
used for INSERT,UPADATE ,DELETE
only one exclusive lock at a time
blocks both reads(locking reads) and writes.

3. Intent Lock
----------------
An Intent Lock is a table-level lock that indicates a transaction intends to place row-level locks.
It helps MySQL quickly determine whether a table-level lock conflicts with existing row locks.
Users do not usually create Intent locks manually. InnoDB manages them automatically.

Real-life example:
Imagine a library:
Shared Lock- Many students can read the same book at the same time.
Exclusive Lock - One student borrows the book, so nobody else can read or borrow it until it is returned. 
Intent lock-The librarian marks that someone is about to read or borrow books from a shelf, helping coordinate access without checking every book individually.
